RGB LED

The RGB LED library is used to drive the SK6818 LEDs on the left and right sides with the ESP-IDF RMT driver to generate the waveforms used and comes from UncleRus' open source ESP-IDF Components library project. The core2foraws_rgb_led_init initialization function or core2foraws_init wrapper function configures the Espressif ESP-IDF RMT driver to communicate with the serially connected SK6818 RGB LEDs. The library provides convenience wrapper functions to the ESP-IDF Components library with preset parameters.

Note
The core2foraws_rgb_led_write function must be called after other methods for the RGB LEDs to write to the LEDs.

To enable the BSP hardware features, open the KConfig menu using the command pio run --environment core2foraws --target menuconfig from within the root of the project directory in your PlatformIO terminal window and go to the menu Component Config --> Core2 for AWS hardware features to set the features you want to enable.

Example

The following example initializes the Core2 for AWS, which initializes a number of enabled features, including the power management unit (PMU) and the RGB_LEDs. It then sets the left LED bar to green and the right LED bar to red.

#include <stdint.h>
#include <esp_log.h>
#include "core2foraws.h"
static const char *TAG = "RGB_LED_EXAMPLE";
void app_main( void )
{
core2foraws_init();
core2foraws_rgb_led_side_color_set( RGB_LED_SIDE_LEFT, 0x00ff00 );
core2foraws_rgb_led_side_color_set( RGB_LED_SIDE_RIGHT, 0xff0000 );
core2foraws_rgb_led_write();
}
