Core2 for AWS IoT Kit BSP 2.0.0
Display

The display screen is driven by an ILI9342C controller over the SPI bus. The Core2 for AWS IoT Kit BSP uses the LVGL library to interact with the ILI9342C display controller via the LVGL ESP32 Driver. For the documentation and API reference of the LVGL library used in this BSP, visit https://docs.lvgl.io/7.11/index.html.

Before using the LVGL APIs to write to the screen, initialize the display with core2foraws_display_init or the core2foraws_init wrapper function first. When performing any operations to the screen after initialization, the core2foraws_common_spi_semaphore FreeRTOS semaphore must be taken before calling the LVGL APIs, and released after in order to prevent a race condition when writing to the screen.

To enable the BSP hardware features, open the KConfig menu using the command pio run --environment core2foraws --target menuconfig from within the root of the project directory in your PlatformIO terminal window and go to the menu Component Config --> Core2 for AWS hardware features to set the features you want to enable.

Example

The following example initializes the Core2 for AWS, which initializes a number of enabled features, including the power management unit (PMU) and the display controller over the SPI bus. It then creates a LVGL label widget, sets the text of the label to "Hello World!", and aligns the label to the center of the screen.

#include <stdint.h>
#include <freertos/FreeRTOS.h>
#include <freertos/semphr.h>
#include "core2foraws.h"
void app_main( void )
{
core2foraws_init();
xSemaphoreTake( core2foraws_common_spi_semaphore, portMAX_DELAY );
lv_obj_t *hello_label = lv_label_create( lv_scr_act(), NULL );
lv_label_set_text( hello_label, "Hello World" );
lv_obj_align( hello_label, NULL, LV_ALIGN_CENTER, 0, 0 );
xSemaphoreGive( core2foraws_common_spi_semaphore );
}
