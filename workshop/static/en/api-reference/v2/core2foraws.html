Core2 for AWS IoT Kit BSP 2.0.0
Core2forAWS

The Core2 for AWS wrapper library simply provides a convenience function to initalize some enabled BSP supported hardware features. These features include the power management, display, motion sensor, real-time clock, hardware cryptographic secure element, side RGB LEDs, and virtual button drivers. It also starts the Wi-Fi provisioning over BLE. Other features such as the microphone, speaker, SD card, and Wi-Fi must be manually initialized as needed. If the feature is not enabled, it will not get initialized. If the function is called and the entire BSP is disabled, it will log an error to the serial output.

To enable the BSP hardware features, open the KConfig menu using the command pio run --environment core2foraws --target menuconfig from within the root of the project directory in your PlatformIO terminal window and go to the menu Component Config --> Core2 for AWS hardware features to set the features you want to enable.

Example

The following example initializes Core2 for AWS, which has the power and display features enabled in the firmware configuration, and adjusts the screen backlight brightness to full bright (100%):

#include <stdio.h>
#include <esp_log.h>
#include "core2foraws.h"
static const char *TAG = "MAIN";
void app_main( void )
{
core2foraws_init(); // Initializes enabled hardware features (except mic, speaker, SD).
core2foraws_power_backlight_set( 100 );
}
